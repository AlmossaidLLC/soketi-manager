Pusher SDK Compatible

Use It in Your Code

Soketi Manager is 100% compatible with Pusher SDKs. If you're already using Pusher, just change the host. If you're starting fresh, pick your language and copy-paste.

Server-Side — Trigger Events 

import Pusher from "pusher";

// Initialize the Pusher server SDK pointing to your Soketi instance
const pusher = new Pusher({
  appId: "my-chat-app",          // Your app ID from the Manager UI
  key: "app-key",
  secret: "app-secret",
  host: "your-server.com",       // Your Soketi Manager host
  port: "6001",
  useTLS: false,
  cluster: "mt1",               // Required but unused for Soketi
});

// Trigger an event on a channel
await pusher.trigger("notifications", "order-placed", {
  orderId: "ORD-12345",
  message: "Your order has been placed!",
  amount: 59.99,
});

// Trigger to multiple channels at once
await pusher.trigger(
  ["notifications", "admin-dashboard"],
  "new-user",
  { username: "john_doe", plan: "premium" }
);

Client-Side — Listen for Events 

import Pusher from "pusher-js";

// Connect to your Soketi instance
const pusher = new Pusher("app-key", {
  wsHost: "your-server.com",
  wsPort: 6001,
  forceTLS: false,
  disableStats: true,
  enabledTransports: ["ws", "wss"],
  cluster: "mt1",
});

// Subscribe to a channel
const channel = pusher.subscribe("notifications");

// Listen for events
channel.bind("order-placed", (data) => {
  console.log("New order:", data.orderId);
  showNotification(data.message);
});

// Listen for multiple events on the same channel
channel.bind("new-user", (data) => {
  console.log("New user registered:", data.username);
});

// Connection state management
pusher.connection.bind("state_change", ({ current }) => {
  console.log("Connection state:", current);
});
import { useEffect, useState } from "react";
import Pusher from "pusher-js";

function useChannel(channelName) {
  const [messages, setMessages] = useState([]);

  useEffect(() => {
    const pusher = new Pusher("app-key", {
      wsHost: "your-server.com",
      wsPort: 6001,
      forceTLS: false,
      disableStats: true,
      enabledTransports: ["ws", "wss"],
      cluster: "mt1",
    });

    const channel = pusher.subscribe(channelName);

    channel.bind("order-placed", (data) => {
      setMessages((prev) => [...prev, data]);
    });

    return () => {
      channel.unbind_all();
      pusher.unsubscribe(channelName);
      pusher.disconnect();
    };
  }, [channelName]);

  return messages;
}

// Usage in a component
function OrderNotifications() {
  const orders = useChannel("notifications");

  return (
    <div>
      {orders.map((order, i) => (
        <div key={i}>Order {order.orderId}: {order.message}</div>
      ))}
    </div>
  );
}
// composables/usePusher.js
import { ref, onMounted, onUnmounted } from "vue";
import Pusher from "pusher-js";

export function usePusher(channelName, eventName) {
  const messages = ref([]);
  let pusher, channel;

  onMounted(() => {
    pusher = new Pusher("app-key", {
      wsHost: "your-server.com",
      wsPort: 6001,
      forceTLS: false,
      disableStats: true,
      enabledTransports: ["ws", "wss"],
      cluster: "mt1",
    });

    channel = pusher.subscribe(channelName);
    channel.bind(eventName, (data) => {
      messages.value.push(data);
    });
  });

  onUnmounted(() => {
    channel?.unbind_all();
    pusher?.unsubscribe(channelName);
    pusher?.disconnect();
  });

  return { messages };
}

Migrating from Pusher?

It's as simple as changing your connection config. Replace host: "api-xx.pusher.com" with your Soketi Manager host. All your existing channel subscriptions, event bindings, and broadcasting logic will work without any changes. Soketi implements the full Pusher protocol — channels, presence, private channels, client events, and webhooks are all supported.

Quick Start

Get up and running in under 2 minutes

1

Pull the Image

Get the latest pre-built Docker image from GitHub Container Registry.

docker pull ghcr.io/almossaidllc/soketi-manager:latest
2

Run the Container

Start with a single command. Config auto-initializes.

docker run -d \ --name soketi-app-manager \ -p 3000:3000 \ -p 6001:6001 \ ghcr.io/almossaidllc/soketi-manager:latest
3

Create Your First App

Open the Manager UI, click "Create App", and you're ready to go.

Open Manager Open Playground

Need to Persist Config?

Use a Docker volume to keep your configuration across container restarts:

docker run -d \ --name soketi-app-manager \ -p 3000:3000 \ -p 6001:6001 \ -v soketi-config:/app/config:rw \ -e SOKETI_CONFIG_FILE=/app/config/soketi.json \ ghcr.io/almossaidllc/soketi-manager:latest
